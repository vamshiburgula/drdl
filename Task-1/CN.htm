Aspect

Description

Purpose

View and configure network interfaces on Unix-like systems.

Usage

ifconfig [interface] [options]

Functionality

      Displays interface details such as name, IP address, MAC address, etc.

Configures interfaces by assigning IP addresses, bringing interfaces up or down, etc.

Options

-a: Display information about all interfaces, including those that are down.

up/down: Bring an interface up or down, respectively.

netmask, broadcast, etc.: Configure specific parameters.

Deprecation

Some Linux distributions have deprecated ifconfig in favor of the ip command.

Ping Command

Aspect

Description

Purpose

Test the reachability of a host on an IP network and measure round-trip time.

Syntax

ping [options] destination

Usage

destination: Specifies the host or IP address to ping.

Options can be used to customize behavior.

Functionality

Sends ICMP echo request packets to the specified destination.

Waits for ICMP echo reply packets from the destination.

Measures round-trip time (RTT).

Options

-c count: Specifies the number of packets to send.

-i interval: Specifies the interval between sending packets.

-s packetsize: Specifies the size of the ICMP packets to send.

-t timeout: Specifies the maximum time to wait for a reply.

-q: Quiet output, only displays summary statistics.

-v: Verbose output, displays detailed information about each packet.

Output

Displays RTT, packet loss percentage, and other statistics for each packet sent.

Provides a summary of statistics after completion.

 

ping is a versatile command commonly used for network diagnostics and performance testing.

nslookup  Command

Aspect

Description

Purpose

Query DNS servers to obtain domain name or IP address mapping, DNS record types, and related information.

Syntax

nslookup [options] [hostname or IP address]

Usage

[hostname or IP address]: Specifies the domain name or IP address to be looked up.

Options can be used to customize behavior.

Functionality

Queries DNS servers to obtain various types of information about domain names and IP addresses.

Common uses include looking up IP addresses associated with domain names and performing reverse DNS lookups.

Options

-type=rrtype: Specifies the type of DNS record to query (e.g., A, MX, NS, PTR, etc.).

-query=type: Equivalent to -type.

-timeout: Specifies the timeout for DNS queries.

-port: Specifies the port number to use for the DNS query.

-server: Specifies the DNS server to query.

 

 

Sample Output:

Server:     192.168.1.1

Address:    192.168.1.1#53

Non-authoritative answer:

Name:       example.com

Address:    93.184.216.34

 

OUTPUT DESCRIPTION:

  • Server: Indicates the DNS server used for the lookup, along with its IP address and port number.
  • Non-authoritative answer: Indicates that the response is obtained from a DNS server other than the authoritative server for the domain.
  • Name: The domain name being looked up.
  • Address: The IP address associated with the domain name

netstat  Command

Aspect

Description

Purpose

Provides information about network connections and networking statistics.

Syntax

netstat [options]

Usage

Options can be used to specify what information to display or customize the output.

Functionality

Displays information about active network connections, routing tables, interface statistics, etc.

Options

-a

-t

-u

-n

-p

-r

-s

Output

Includes information such as protocol, local and remote addresses, state of the connection, PID of the process, and more.

 

 

SAMPLE  OUTPUT:

         Proto Recv-Q Send-Q Local Address           Foreign Address         State

tcp        0      0 0.0.0.0:22              0.0.0.0:*               LISTEN

tcp        0      0 192.168.1.2:443         203.0.113.10:54124      ESTABLISHED

tcp        0      0 192.168.1.2:587         203.0.113.11:52341      TIME_WAIT

udp        0      0 0.0.0.0:68              0.0.0.0:*

 

OUTPUT DESCRIPTION:

  • Proto: Protocol of the connection (tcp or udp).
  • Recv-Q: Receive queue length.
  • Send-Q: Send queue length.
  • Local Address: IP address and port number of the local endpoint.
  • Foreign Address: IP address and port number of the remote endpoint.
  • State: State of the connection (e.g., LISTEN, ESTABLISHED, TIME_WAIT).
